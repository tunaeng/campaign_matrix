Введение
Добро пожаловать в документацию REST API для работы с организациями. Этот API предоставляет возможность получения информации об организациях, их типах, регионах, сферах деятельности и связанных проектах.

Особенности API: Поддерживает расширенную фильтрацию по различным параметрам, включая вложенные связи (регионы, федеральные округа, проекты).
1. Аутентификация
Для доступа к API требуется токен.

В админ‑панели Django перейдите в раздел "Токены".
Создайте новый токен для нужного пользователя.
Сохраните токен для будущего использования.
2. Основной эндпоинт
2.1 Получение данных об организациях
Эндпоинт:	/contacts/api/organizations/
Метод HTTP:	GET
Аутентификация:	Требуется (токен в заголовке Authorization)
Пагинация:	Поддерживается (стандартная Django REST)
2.2 Формат запроса
Заголовки обязательные:

Заголовок	Значение	Описание
Authorization	Token YOUR_TOKEN	Токен аутентификации
Accept	application/json	Формат ответа (рекомендуется)
2.3 Примеры использования
Пример 1: JavaScript (Fetch API)
// Получение всех организаций
fetch('https://bitrix24.tuna-edu.ru/contacts/api/organizations/', {
    method: 'GET',
    headers: {
        'Authorization': 'Token 4a8389aa7f015d3a68e44e3f4bae94d63c7ac097',
        'Accept': 'application/json'
    }
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Ошибка:', error));
Пример 2: cURL
curl -X GET 'https://bitrix24.tuna-edu.ru/contacts/api/organizations/' \
            -H 'Authorization: Token 4a8389aa7f015d3a68e44e3f4bae94d63c7ac097' \
            -H 'Accept: application/json'
Пример 3: Python (Requests)
import requests
headers = {
    'Authorization': 'Token 4a8389aa7f015d3a68e44e3f4bae94d63c7ac097',
    'Accept': 'application/json'
}
response = requests.get('https://bitrix24.tuna-edu.ru/contacts/api/organizations/', headers=headers)
data = response.json()
print(data)
3. Фильтрация результатов
API поддерживает расширенную фильтрацию по различным параметрам организаций.

3.1 Доступные фильтры
Фильтр	Параметр	Описание	Тип поиска	Примечание
Тип организации	type	Тип организации (например, "РОИВ")	Точное совпадение	-
Дата создания (содержит)	date	Поиск по части даты создания	Частичное совпадение	Формат: YYYY-MM-DD
Сфера деятельности	prof_activity	Сфера деятельности	Точное совпадение	Только для РОИВ
Сфера деятельности (содержит)	prof_activity__contains	Поиск по части названия сферы деятельности	Частичное совпадение	Только для РОИВ
Проект	project	Поиск организаций по проектам	Частичное совпадение	По названию проекта
Регион	region	Название региона	Точное совпадение	-
Регион (содержит)	region__contains	Поиск по части названия региона	Частичное совпадение	-
Федеральный округ	fed_district	Название федерального округа	Точное совпадение	-
Федеральный округ (содержит)	fed_district__contains	Поиск по части названия фед. округа	Частичное совпадение	-
Федеральная компания	federal	Фильтр по федеральным компаниям	Булевый	true/false
Активность	is_active	Фильтр по активности организации	Булевый	true/false
3.2 Синтаксис фильтрации
Фильтры передаются как query-параметры в URL:

/contacts/api/organizations/?filter1=value1&filter2=value2
3.3 Примеры фильтрации
Пример 1: Поиск по типу организации
https://bitrix24.tuna-edu.ru/contacts/api/organizations/?type=РОИВ
Пример 2: Поиск по региону и федеральному статусу
https://bitrix24.tuna-edu.ru/contacts/api/organizations/?region__contains=Москва&federal=true
Пример 3: Поиск организаций с проектом
https://bitrix24.tuna-edu.ru/contacts/api/organizations/?project=Цифровая%20трансформация
Пример 4: Комбинированный поиск для РОИВ
https://bitrix24.tuna-edu.ru/contacts/api/organizations/?type=РОИВ&prof_activity__contains=образование&fed_district=Центральный
Обратите внимание: Фильтры prof_activity и prof_activity__contains работают только при установленном параметре type=РОИВ.

4.2 Структура данных в ответе
Поле	Тип	Описание
name	String	Краткое название организации
full_name	String	Полное официальное название
type	String	Тип организации
region	String	Регион расположения
federal_company	Boolean	Является ли федеральной компанией
fed_district	String	Федеральный округ
prof_activity	String	Сфера деятельности (для РОИВ)
projects	Array	Список проектов организации
is_active	Boolean	Активна ли организация
created_at	String (ISO 8601)	Дата создания записи
updated_at	String (ISO 8601)	Дата последнего обновления
5. Коды ответов и ошибки
5.1 Успешные коды ответов
Код	Описание
200	OK — Запрос выполнен успешно
5.2 Коды ошибок
Код	Описание	Решение
400	Bad Request — Неверный формат запроса	Проверьте синтаксис URL и параметры фильтрации
401	Unauthorized — Требуется аутентификация	Убедитесь, что передали корректный токен
403	Forbidden — Доступ запрещен	Ваш токен не имеет прав на доступ к API
404	Not Found — Ресурс не найден	Проверьте правильность URL эндпоинта
500	Internal Server Error — Ошибка сервера	Попробуйте позже или обратитесь к администратору
5.3 Примеры ошибок
Ошибка 401: Отсутствует токен
{
    "detail": "Authentication credentials were not provided."
}
Решение: Добавьте заголовок Authorization с вашим токеном.

Особенности API: Все эндпоинты требуют аутентификации по токену. Данные возвращаются в формате JSON.

Описание:	Возвращает список всех федеральных округов с вложенными списками регионов

Эндпоинт:	/contacts/api/get_all/region/

Описание:	Возвращает список всех регионов

Authorization	Token YOUR_TOKEN	Токен аутентификации
