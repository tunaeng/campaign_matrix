# Импорт востребованности профессий

Инструкция по загрузке данных о востребованности профессий по регионам для выбранного федерального оператора.

---

## Кто может импортировать

Импорт доступен только пользователям с ролью **Администратор**. На странице «Матрица востребованности» у таких пользователей отображается кнопка **«Импорт»**.

---

## Как начать импорт

1. Откройте раздел **«Матрица востребованности»** (меню слева).
2. Нажмите кнопку **«Импорт»** (справа в блоке со статистикой или под фильтрами).
3. Откроется окно **«Импорт востребованности»** с пошаговым мастером.

---

## Шаг 1. Загрузка файла

1. **Выберите федерального оператора** — данные будут загружены для этого оператора.
2. **Выберите год** импорта (2024, 2025 или 2026).
3. Нажмите **«Проверить файл»** и выберите файл на компьютере.

Поддерживаются форматы: **CSV** (разделитель — точка с запятой или запятая) и **XLSX** (Excel).  
Рекомендуемая кодировка для CSV: **UTF-8 с BOM** (удобно открывать и сохранять в Excel).

После загрузки система проверит файл и при необходимости предложит следующие шаги: сопоставление регионов и/или профессий.

---

## Форматы файла

### Формат «Матрица» (рекомендуется)

Подходит для таблицы: **строки — профессии, столбцы — регионы**, в ячейках — **1** (востребована) или **0** (не востребована).

**Структура:**

| Колонка 1 | Регион 1 | Регион 2 | … |
|-----------|----------|----------|---|
| Название колонки профессий | Название региона | Название региона | … |
| Профессия 1 | 1 | 0 | … |
| Профессия 2 | 0 | 1 | … |

- **Первая строка** — заголовок: в первой ячейке может быть подпись вроде «Наименование направлений подготовки, специальностей, профессий», в остальных — **точные названия регионов** из справочника системы.
- **Со второй строки** — данные: в первой колонке **название профессии**, в остальных — **1** или **0** (в Excel могут отображаться как 1.0 и 0.0 — это допустимо).

Названия регионов должны совпадать с теми, что есть в системе (регистр не учитывается). Примеры: «г. Москва», «Московская область», «Ханты-Мансийский АО - Югра».

---

### Формат «Построчный»

Каждая строка — одна связь «профессия – регион – востребована».

**Колонки (например, через точку с запятой):**

- `код_профессии` (или `number`) — номер профессии по перечню (необязательно).
- `профессия` (или `name`, `профессия`) — название профессии.
- `регион` (или `region`, `region_name`) — название региона.
- `востребована` (или `demanded`, `is_demanded`) — признак: **1** или **да** = востребована, **0** или **нет** = не востребована.
- `год` (или `year`) — год (необязательно, подставляется год из шага 1).

---

## Значения «востребована» / «не востребована»

**Считается «востребована» (да):**  
`1`, `1.0`, `да`, `д`, `true`, `yes`, `y`, `x`, `+`

**Считается «не востребована» (нет):**  
`0`, `0.0`, `нет`, `н`, `false`, `no`

Любое другое значение трактуется как «не востребована».

---

## Шаг 2. Регионы (если появился)

Если в файле есть названия регионов, которых нет в справочнике системы (опечатка, другое написание, старое название), мастер покажет их списком.

Для каждой такой записи нужно **выбрать замену** — соответствующий регион из выпадающего списка. Поиск в списке идёт по названию (например, «Москва» найдёт «г. Москва»).

После того как все нераспознанные регионы будут сопоставлены, нажмите **«Далее»**.

---

## Шаг 3. Профессии (если появился)

Если в файле есть профессии, которых ещё нет в справочнике, мастер предложит для каждой выбрать:

- **Добавить как новую** — профессия будет создана в системе.
- **Заменить на существующую** — выбрать из списка уже существующую профессию, к которой привяжут данные из файла.

Заполните выбор для всех строк и нажмите **«Далее»**.

---

## Шаг 4. Подтверждение

На последнем шаге отображается сводка:

- формат файла (матрица или построчный);
- сколько новых профессий будет создано;
- сколько связей будет создано и обновлено;
- при необходимости — таблица замен регионов и решений по новым профессиям.

Проверьте данные и нажмите **«Применить импорт»**. После успешного импорта окно закроется, матрица на странице обновится.

---

## Шаблоны файлов

В проекте есть готовые шаблоны для заполнения:

- **Матрица (CSV):** `backend/data/demand_import_templates/demand_matrix_import_template_wide.csv`
- **Матрица (Excel):** `backend/data/demand_import_templates/demand_matrix_import_template_wide.xlsx`
- **Построчный (CSV):** `backend/data/demand_import_templates/demand_matrix_import_template_row.csv`

В шаблонах матрицы уже подставлены актуальные названия регионов из системы. Можно скопировать нужный файл, заполнить профессии и значения 1/0 и загрузить через мастер импорта.

При необходимости шаблоны можно заново сгенерировать (с актуальным списком регионов) командой:

```bash
python manage.py generate_demand_import_templates
```

Опционально можно указать папку для сохранения: `--output-dir /путь/к/папке`.

## Частые вопросы

**Импорт не запускается / нет кнопки «Импорт».**  
Импорт доступен только пользователям с ролью «Администратор». Обратитесь к администратору системы для назначения роли или выполнения импорта.

**В списке регионов не находится нужный регион при поиске.**  
Убедитесь, что загружен полный список регионов (система запрашивает до 200 записей). Если регион есть в справочнике, он должен отображаться в выпадающем списке при вводе части названия.

**Файл в формате матрицы не распознаётся.**  
Проверьте, что первая строка — заголовок с названиями регионов начиная со второй ячейки, а со второй строки идут данные: профессия в первой колонке, далее 1 или 0 по регионам. Названия регионов должны совпадать со справочником (можно свериться с шаблоном).

**После импорта данные не обновились.**  
Обновите страницу (F5) или закройте и снова откройте «Матрица востребованности». Убедитесь, что выбран тот же федеральный оператор и год, для которых выполнялся импорт.
